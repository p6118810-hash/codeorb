# CodeOrb App Workspace Guide

## 重要说明

当前 `app/` 目录为空，属于预留子项目目录。

在这个目录工作时请遵守：
- 不要凭空假设这里已经有 Web、桌面端或服务端实现
- 不要在没有明确任务的情况下初始化框架、安装依赖或生成模板代码
- 如果用户要求在 `app/` 下创建新项目，请保持文件全部收敛在本目录中

## Workspace Status

`app/` 目前没有源码、配置或脚本文件，可视为待规划区域。

## Recommended Use

如果未来需要使用本目录，推荐承载以下其中一种职责：
- 新的独立客户端应用
- 与 `mobile/` 配套的共享 App 项目
- 演示壳层、包装器或实验性实现

正式开始实现前，建议先明确：
- 技术栈
- 入口文件位置
- 依赖管理方式
- 与 `frontend/`、`mobile/`、`backend/` 的边界

## Documentation Rules

工作区级文档建议统一放在根目录：
- `docs/` - 设计、架构、任务说明
- `scripts/` - 自动化脚本
